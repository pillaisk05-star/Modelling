# Architectural Decision Records (ADR)

This directory contains the Architectural Decision Records (ADRs) for the Modelling project. Each ADR is a document that captures a particular architectural decision made in the project. They serve as a historical record, providing context and rationale for architectural choices.

## How to Create an ADR

1. Create a new file for your ADR in this directory following the naming convention `adr-<number>.md`.
2. Include the following sections:
   - **Title**
   - **Status** (Proposed, Accepted, Deprecated, etc.)
   - **Context**
   - **Decision**
   - **Consequences**
3. Submit a pull request to have your ADR reviewed.
